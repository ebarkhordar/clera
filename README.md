# Clera

[![Python 3.12+](https://img.shields.io/badge/python-3.12%2B-blue.svg)](https://www.python.org/)

[![pre-commit](https://img.shields.io/badge/pre--commit-enabled-brightgreen.svg)](https://pre-commit.com/)

**Clera** is a personal secretary bot for Telegram. It reads the messages your
contacts send you and drafts replies **in your voice** — then you approve each
one with a single tap before it's sent as you.

Built on Telegram's [Business connection](https://core.telegram.org/bots/features#secretary-bots)
API, with replies generated by Claude.

> **Draft-first by design.** Nothing is ever sent without your explicit approval.
> Auto-reply is strictly opt-in and limited to contacts you allowlist.

---

## How it works

```
A contact messages you
        │  business_message update
        ▼
   Policy check ──────────────►  allowlist · active hours
        │
        ▼
   Claude drafts a reply  ────►  token usage metered → estimated cost
        │
        ▼
   Draft sent to your control chat   [ ✅ Send ]   [ 🗑 Discard ]
        │  you tap Send
        ▼
   sendMessage(…, business_connection_id)  ──►  delivered as you
```

## Features

- **Reads and answers your real chats**, not a separate bot conversation, via
  Telegram Business connections.
- **Draft-first approval** — every reply waits for your tap in a private control
  chat. Optional allowlisted auto-reply for trusted contacts.
- **Claude-powered drafts** with a selectable model tier (a fast, cheaper model
  or a top-tier one).
- **Pluggable LLM providers** behind a small interface (Claude today; others
  drop in).
- **Runs without secrets** — with no API key it produces a safe placeholder
  draft, so you can exercise the whole flow before wiring anything up.
- **Safety controls**: contact allowlist, active-hours window, and a hard
  discard.

## Requirements

- Python 3.12+
- A bot token from [@BotFather](https://t.me/BotFather)
- **Telegram Premium** on the account you connect. Business connections are a
  Premium feature; there is no free-tier equivalent.
- *(Optional)* an `ANTHROPIC_API_KEY` for real drafts.

## Quick start

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env      # then set TELEGRAM_BOT_TOKEN
python -m app.doctor      # pre-flight: validates your token and config
python -m app.main        # long-polling; no public URL required
```

Then, in the Telegram app, connect the bot to your account:
**Settings → Business → Chatbots → select your bot**.

For a full walkthrough against a live account, see
[docs/LIVE_TEST.md](docs/LIVE_TEST.md).

## Configuration

All configuration is via environment variables (loaded from `.env`). See
[`.env.example`](.env.example) for the full list.

| Variable | Required | Description |
| --- | --- | --- |
| `TELEGRAM_BOT_TOKEN` | Yes | Bot token from @BotFather. |
| `CONTROL_CHAT_ID` | No | Your numeric Telegram id, where drafts are sent for approval. Falls back to the owner's own chat. |
| `ANTHROPIC_API_KEY` | No | Enables real Claude drafts. Without it, a placeholder draft is used. |
| `MODEL_BEST` / `MODEL_FAST` | No | Model ids for the two tiers. |
| `DEFAULT_TIER` | No | `best` or `fast`. |
| `COST_MARKUP` | No | Multiplier applied to the estimated token cost shown per draft. |

## Project structure

```
app/
  main.py          Entrypoint — registers handlers, requests business_* updates
  doctor.py        Pre-flight connectivity check
  config.py        Environment-driven settings
  handlers/        Telegram handlers (business connection, control chat, managed)
  agent/           LLM provider interface, Claude implementation, prompts
  billing/         Token pricing and cost estimation
  policy/          Allowlist, active-hours, draft-vs-auto rules
  store/           Data models + storage (SQLite by default, or in-memory)
tests/             Test suite
docs/              Design notes and the live-test runbook
```

## Development

```bash
pip install -r requirements.txt ruff pre-commit
pre-commit install            # run the hooks on every commit

ruff check . && ruff format --check .
pytest -q
pre-commit run --all-files
```

CI runs Ruff, the test suite, and a [gitleaks](https://github.com/gitleaks/gitleaks)
secret scan on every push and pull request
(see [`.github/workflows/ci.yml`](.github/workflows/ci.yml)).

## Security

Please report vulnerabilities privately — see [SECURITY.md](SECURITY.md). This
project handles private messages and access tokens, so never commit secrets;
keep them in the gitignored `.env`.
