# Clera

[![License: MIT](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Python 3.12+](https://img.shields.io/badge/python-3.12%2B-blue.svg)](https://www.python.org/)
[![Code style: Ruff](https://img.shields.io/badge/code%20style-ruff-261230.svg)](https://github.com/astral-sh/ruff)
[![pre-commit](https://img.shields.io/badge/pre--commit-enabled-brightgreen.svg)](https://pre-commit.com/)

**Clera** is a personal secretary for Telegram. It reads the messages your
contacts send you and **answers them automatically, in your voice**, using your
full conversation history with each contact. When a message needs *you* —
money, commitments, things only you know — it doesn't reply; it pings you
instead.

Built on Telegram's [Business connection](https://core.telegram.org/bots/features#secretary-bots)
API, with replies generated by Claude. The direction is a hosted platform where
each client gets **their own bot** in a few taps (Telegram managed bots) and
pays per usage — see [docs/PLATFORM.md](docs/PLATFORM.md).

> **Automatic, with an escalation path.** For every message the agent decides:
> reply as you, stay silent, or hand it to you personally. It never invents
> facts or agrees to payments/commitments — those always come to you.

---

## How it works

```
A contact messages you
        │  business_message update
        ▼
   Policy check ─────────────►  enabled · hours · optional allowlist
        │
        ▼
   Claude reads the thread history + contact profile, then decides:
        │
        ├── reply   ──► sendMessage(…, business_connection_id) — sent as you
        ├── silent  ──► nothing (conversation over, or you're handling it)
        └── notify  ──► "👋 this one needs you" in your control chat
```

## Features

- **Reads and answers your real chats**, not a separate bot conversation, via
  Telegram Business connections.
- **Fully automatic** — for each message the agent replies in your voice,
  stays silent, or escalates to you (money, commitments, things only you
  know). A draft-approval review mode remains available as an opt-in.
- **Per-contact memory** — thread history, an LLM-maintained contact profile,
  and your own typed messages teach it your voice. Start in collect-only mode,
  then `python -m app.backfill` builds profiles from the gathered history.
- **Hears and sees** — voice notes are transcribed locally (mlx-whisper, no
  API), photos are described via Claude vision, so the secretary isn't blind
  to half of a real conversation.
- **You stay in control from Telegram** — `/status`, `/auto`, `/review`,
  `/pause`, `/mute <name>`, and a **daily digest** of everything it did.
- **Managed-bot provisioning** — `python -m app.platform` runs a manager bot
  that creates a personal secretary bot for each client (they own it, we
  operate it) plus a runner per client bot.
- **Deployable in one command** — Docker, launchd, or systemd recipes
  ([docs/DEPLOY.md](docs/DEPLOY.md)); a single-instance lock prevents the
  classic two-pollers-one-token conflict.
- **Claude-powered** with a selectable model tier, behind a pluggable provider
  interface; token usage metered per reply. With no API key it never
  auto-sends — placeholder output is escalated to you instead.

## Requirements

- Python 3.12+
- A bot token from [@BotFather](https://t.me/BotFather)
- **Telegram Premium** on the account you connect — believed required for
  Business connections, though the current bot-features docs no longer state
  it (verification tracked in [docs/PLATFORM.md](docs/PLATFORM.md)).
- *(Optional)* an `ANTHROPIC_API_KEY` for real replies (without it nothing is
  auto-sent).

## Quick start

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env      # then set TELEGRAM_BOT_TOKEN
python -m app.doctor      # pre-flight: validates your token and config
python -m app.main        # single-bot mode: your token IS the secretary
```

Then, in the Telegram app, connect the bot to your account:
**Settings → Business → Chatbots → select your bot**.

To run the **platform** instead (manager bot that provisions a personal
secretary bot per client and operates the whole fleet — requires Bot
Management Mode enabled on the token in BotFather):

```bash
python -m app.platform    # manager + one runner per managed bot
```

For a full walkthrough against a live account, see
[docs/LIVE_TEST.md](docs/LIVE_TEST.md).

## Owner commands (in your control chat)

| Command | Effect |
| --- | --- |
| `/status` | Mode, last-24h activity, spend, muted contacts |
| `/auto` / `/review` | Automatic replies ↔ approve-every-draft |
| `/pause` / `/resume` | Stop/start acting (recording continues) |
| `/mute <name>` / `/unmute <name>` | Never act in one chat |
| `/digest` | On-demand summary of the last 24h (also sent daily) |

Recommended rollout: `COLLECT_ONLY=true` for a few days → `python -m
app.backfill` → restart in review mode → `/auto` once you trust it. Full
deployment recipes (Docker/launchd/systemd): [docs/DEPLOY.md](docs/DEPLOY.md).

## Configuration

All configuration is via environment variables (loaded from `.env`). See
[`.env.example`](.env.example) for the full list.

| Variable | Required | Description |
| --- | --- | --- |
| `TELEGRAM_BOT_TOKEN` | Yes | Bot token from @BotFather (in platform mode: the manager bot's token). |
| `CONTROL_CHAT_ID` | No | Single-bot dev override: numeric Telegram id for escalations/activity. Defaults to the connection owner's own chat (leave unset in platform mode). |
| `ANTHROPIC_API_KEY` | No | Enables real Claude replies. Without it, nothing is auto-sent — messages are escalated to you. |
| `NOTIFY_AUTO_REPLIES` | No | Echo each auto-sent reply to your control chat (default `true`). |
| `MODEL_BEST` / `MODEL_FAST` | No | Model ids for the two tiers. |
| `DEFAULT_TIER` | No | `best` or `fast`. |
| `COST_MARKUP` | No | Multiplier applied to the estimated token cost metered per reply. |

## Project structure

```
app/
  main.py          Single-bot entrypoint — registers handlers, business_* updates
  platform.py      Platform entrypoint — manager bot + a runner per managed bot
  doctor.py        Pre-flight connectivity check
  config.py        Environment-driven settings
  handlers/        Telegram handlers (business, control chat, managed provisioning)
  agent/           LLM provider interface, Claude implementation, prompts
  billing/         Token pricing and cost estimation
  policy/          Engagement rules: enabled, hours, allowlist, auto-vs-review
  store/           Data models + storage (SQLite by default, or in-memory)
tests/             Test suite
docs/              Design notes, platform direction, live-test runbook
```
